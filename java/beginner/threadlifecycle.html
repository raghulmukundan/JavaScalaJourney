Thread Lifecycle & States

A thread can be in one of the following states:

  • New: A thread that has been created but not yet started.

  • Blocked: A thread that is blocked and waiting for a monitor lock.

  • Runnable: A thread that is ready to run and is waiting for CPU time.

  • Waiting: A thread that is waiting indefinitely for another thread to perform a particular action.

  • Timed Waiting: A thread that is waiting for another thread to perform an action for up to a specified waiting time.

  • Terminated: A thread that has exited.

In-Depth Explanation

  • Thread Creation:When a new thread is created using new Thread(), the JVM allocates memory and resources for the thread.
  • start() Method:This native method creates a new OS-level thread and schedules it to run the run() method.
  • sleep() Method: It temporarily pauses the thread's execution. The thread remains in the Timed Waiting state during this period.
  • join() Method: It allows one thread to wait for the completion of another. It internally uses the wait() method of the Object class.
  • yield() Method: It hints to the thread scheduler that the current thread is willing to yield its current use of a processor but it's not guaranteed.
